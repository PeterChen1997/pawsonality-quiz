# 爪格实验室：宠物人格测试数据说明

> 本文档说明本项目如何基于 15 维向量进行匹配，以及当前这版双题库重构后的数据结构。

## 项目定位

本项目基于 `pingfanfan/SBTI` 二次开发，保留了：
- 纯前端静态部署方式
- 15 维度 + 曼哈顿距离匹配逻辑
- 特殊人格彩蛋机制

重构后重点改成：
- 面向中文互联网传播的 **宠物 MBTI / 狗格测试 / 猫格测试**
- 更适合女性主理人做营销验证的轻内容语气
- 可配置的主理人 / 品牌露出位
- 页面内明确标注源码来源
- 猫 / 狗两套独立题库，不再共用同一套主问题
- 结果页使用更完整的 `subtitle / footnote / deepDive / tags` 文案层

## 评分模型

### 1. 维度结构

共 15 个维度，分为 5 个模型：

1. 本体气场
2. 情感亲密
3. 生活态度
4. 行动节奏
5. 社交边界

每个物种各自有一套主问题库：

- 狗狗：12 道主问题
- 猫猫：12 道主问题

每道题会同时影响 3 到 4 个维度，题目选项分值仍使用 `1 / 2 / 3` 语义，只是改为按维度取平均分。

### 2. 分数计算

- 每道题会对多个相关维度写入 `1 / 2 / 3` 的信号分
- 每个维度汇总自己命中的题目后取平均值
- 分段映射为：
- `1.00 ~ 1.67 => L`
- `1.68 ~ 2.33 => M`
- `2.34 ~ 3.00 => H`

### 3. 人格匹配

把 15 个维度转成一个 `L/M/H` 向量，并与对应物种的人格库中的 pattern 做曼哈顿距离比较：

- 距离越低越相似
- 若距离相同，则优先 exact 命中更多者
- 再按 similarity 排序

### 4. 特殊人格

- 物种内隐藏人格
  - 狗狗：`STAR`
  - 猫猫：`DIVA`
  - 当前为 12 题内的双条件彩蛋触发
- 共享混合人格
  - `MIXI`
  - 当标准人格相似度低于阈值时作为兜底

### 5. 稳定性校验

项目内置 `npm run validate`，用于检查：

- 维度顺序、题库归属、pattern 长度是否一致
- 12 题短测里每个维度是否至少被命中 2 次
- 每个标准人格的 exact pattern 是否能命中自己
- 类型库是否过于拥挤，避免多个结果只差一两个维度

## 品牌化内容结构

### 首页
- 品牌名
- 宠物人格测试副标题
- 主理人信息
- 源码来源说明

### 结果页
- 主人格
- `subtitle / footnote / deepDive / tags`
- 15 维雷达图
- TOP 5 匹配列表
- 主理人露出位
- 源码来源说明
- 品牌化免责声明

### 分享图
- 结果 code + 中文名 + 副标题
- 匹配度
- 15 维雷达 / 条形图
- 主理人露出
- 品牌 footer

## 可继续定制的最关键文件

- `data/config.json`：品牌名、主理人、部署命令、免责声明
- `data/questions.json`：猫狗双题库、特殊题触发逻辑
- `data/types.json`：结果 code、标签、pattern 与解释
- `src/style.css`：视觉风格

## 适用场景

- 小红书互动测试
- 宠物品牌人格化传播
- 主理人人设验证
- 私域引流页
- 女性向轻娱乐内容测试
